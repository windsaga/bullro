# Technology Trends 유입 성장 전략

## 현재 상황

- 블로그는 자동 발행 파이프라인과 기본 SEO 구성이 갖춰져 있다.
- Google/Site Kit, Rank Math, 대표 이미지, 사이트맵 등 기본 인프라는 준비되어 있다.
- 아직 유저 유입이 거의 없는 초기 상태다.
- 검색 유입은 시간이 걸리므로, 초반에는 검색엔진만 기다리지 말고 커뮤니티 배포와 반복 방문 루프를 만들어야 한다.

## 핵심 방향

초기 성장의 우선순위는 `글 생산량`보다 `검색 의도에 맞는 제목`, `직접 배포`, `시리즈화`, `내부 링크`다.

1. 검색형 제목으로 클릭 가능성을 높인다.
2. 글마다 커뮤니티 공유용 요약을 만든다.
3. 로컬 LLM, Codex CLI, NAS 자동화 같은 주제를 시리즈로 묶는다.
4. Reddit, Hacker News, 국내 개발 커뮤니티에 직접 공유한다.
5. Search Console 데이터가 쌓이면 노출 대비 클릭률이 낮은 글부터 제목과 설명을 개선한다.

## 1. 제목을 검색형으로 바꾸기

현재 제목은 기술 리포트 느낌이 강한 편이다. 검색 유입을 노리려면 사용자가 실제로 입력할 문장에 가깝게 바꿔야 한다.

예시:

```text
기존: Qwen 35B-A3B 로컬 추론: 12GB VRAM 최적화 가이드
개선: 12GB VRAM에서 Qwen 35B-A3B 돌리는 방법
개선: RTX 3060 12GB로 실행 가능한 로컬 LLM 설정
```

```text
기존: OpenAI Codex CLI 실전 사용법 — 개발 워크플로우 자동화하기
개선: Codex CLI 설치부터 실제 개발 워크플로우까지
개선: OpenAI Codex CLI로 코드 리뷰와 수정 자동화하기
```

제목 기준:

- 55자 이내
- 핵심 키워드를 앞쪽에 배치
- 모델명, 도구명, 에러명, 하드웨어명처럼 검색 가능한 고유명사를 포함
- “가이드”, “설정”, “비교”, “오류 해결”, “방법” 같은 검색 의도 단어 사용

## 2. 글마다 배포용 요약 만들기

글을 발행한 뒤 커뮤니티에 바로 공유할 수 있는 짧은 요약이 필요하다. 링크만 던지면 반응이 낮다.

각 글마다 다음 형식의 요약을 만든다.

```text
제목:
핵심 요약 3줄:
실무적으로 유용한 점:
주의할 점:
링크:
```

채널별 톤:

- Reddit: 실험 결과, 설정값, 벤치마크, 실패 사례 중심
- Hacker News: 기술적 관찰, 설계 trade-off, 구현 디테일 중심
- X/Twitter: 짧은 핵심 문장과 숫자 중심
- 국내 커뮤니티: 문제 상황, 적용 방법, 비용/환경 중심

## 3. 공유 채널

우선순위가 높은 채널:

- Reddit `r/LocalLLaMA`
- Hacker News
- X/Twitter
- LinkedIn
- OKKY
- 클리앙 개발/AI 게시판
- 디시 AI 관련 갤러리
- Velog, Medium, 브런치 일부 재게시

주의사항:

- 같은 글을 여러 곳에 그대로 복붙하지 않는다.
- 커뮤니티마다 본문 요약을 먼저 제공하고 링크는 뒤에 둔다.
- 홍보성 문장보다 “내가 테스트해보니 이런 결과가 나왔다” 형태가 좋다.
- 해외 커뮤니티에는 한국어 본문 링크만 던지기보다 영어 요약을 함께 제공한다.

## 4. 영어 요약 병행

로컬 LLM, Codex CLI, Qwen, GLM, vLLM 같은 주제는 한국어 검색량이 작다. 해외 커뮤니티 공유를 위해 본문 하단이나 별도 파일로 English summary를 생성한다.

권장 형식:

```text
English Summary
- What this post covers
- Hardware/software environment
- Key benchmark or result
- Main caveat
```

영어 전문을 완전히 별도로 발행하지 않더라도, 공유용 요약만 있어도 Reddit/HN 유입에 도움이 된다.

## 5. 시리즈화

단발 글보다 시리즈가 유입과 체류에 유리하다. 방문자가 다음 글을 누를 이유가 생기고, 내부 링크 SEO도 좋아진다.

시리즈 후보:

```text
로컬 LLM 실험실
1. 12GB VRAM에서 가능한 모델
2. Qwen 35B-A3B 실행
3. vLLM ROCm 설정
4. llama.cpp MTP 성능 비교
5. NAS에서 자동화 파이프라인 돌리기
```

```text
AI 개발도구 워크플로우
1. Codex CLI 실전 사용법
2. Claude Code vs Codex vs Gemini CLI 비교
3. CLI 도구별 보안 모델 비교
4. 실제 PR 리뷰 자동화 패턴
```

```text
AI 블로그 자동화
1. Synology NAS에서 AI 자동화 서버 운영
2. NVIDIA API 기반 글 생성 파이프라인
3. Rank Math SEO 자동화
4. WordPress 발행 자동화와 운영 이슈
```

## 6. Search Console 기반 개선

데이터가 쌓인 뒤에는 Google Search Console을 기준으로 개선한다.

우선순위:

1. 노출은 있는데 클릭이 낮은 글
2. 평균 순위가 8~20위인 글
3. 예상과 다른 검색어로 노출되는 글
4. 클릭이 있는 글과 주제가 가까운 글

개선 액션:

- 제목을 검색어에 맞게 수정
- meta description에 검색어 포함
- 본문 첫 문단에 검색어 자연 삽입
- 관련 글 내부 링크 추가
- 표, 코드, 명령어 섹션 보강

## 7. 기술 글 품질 패턴

기술 블로그는 의견보다 실행 가능한 정보가 유입을 만든다.

잘 먹히는 요소:

- 설치 방법
- 설정 파일
- Docker compose
- benchmark command
- 오류 해결
- 비용 비교
- 하드웨어별 권장 설정
- 직접 테스트한 수치

각 글에 최소 하나 이상 포함하는 것을 목표로 한다.

## 8. 자동화 파이프라인에 추가할 기능

현재 파이프라인은 글 발행 중심이다. 다음 단계는 배포 자동화다.

추가 후보:

- 발행 후 X/Twitter 공유문 생성
- Reddit 공유문 생성
- Hacker News 제목 후보 생성
- 한국 커뮤니티용 짧은 요약 생성
- English summary 생성
- 시리즈 태그 자동 부여
- 관련 글 내부 링크 자동 삽입 강화
- Search Console 쿼리 기반 제목 개선 후보 생성

발행 후 산출물 예시:

```text
posts/share/
  2026-05-09-qwen-35b-a3b.reddit.md
  2026-05-09-qwen-35b-a3b.hn.md
  2026-05-09-qwen-35b-a3b.x.md
  2026-05-09-qwen-35b-a3b.ko-community.md
```
